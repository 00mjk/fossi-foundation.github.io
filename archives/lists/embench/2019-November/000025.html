Hi all,

We have a pull request to make use of 128bit integers if available for
the 64 x 64 -> 128 multiplication in aha-mont64.

  https://github.com/embench/embench-iot/pull/19

On the whole we try to avoid polishing the benchmarks, to ensure as
wide a diversity of coding practice as possible. However there is a
case to be made that this would be the way to write this code if you
have 128-bit integer support, and we do want the benchmarks to
represent good programming practice.

I'd appreciate input from the community on whether we should accept
this pull request.
