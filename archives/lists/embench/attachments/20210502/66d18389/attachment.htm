<p class="MsoNormal"><span style="font-size:12.0pt;font-family:"Times New Roman",serif;color:#0E101A">I’m wondering why we need the warm-up phase at all.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:12.0pt;font-family:"Times New Roman",serif;color:#0E101A">It seems the impact of cold caches/predictors is insignificant.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:12.0pt;font-family:"Times New Roman",serif;color:#0E101A">The impact of warm-up is highest when the whole data is fit in the cache, and the misses are only due to a cold cache.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:12.0pt;font-family:"Times New Roman",serif;color:#0E101A">Therefore, the upper bound of warm-up gain per access is ((Miss-penalty)*(static data size)/(Dynamic accesses))*(base CPI)<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:12.0pt;font-family:"Times New Roman",serif;color:#0E101A"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:12.0pt;font-family:"Times New Roman",serif;color:#0E101A">Warm-up has three parts:<o:p></o:p></span></p>
<ol style="margin-top:0in" start="1" type="1">
<li class="MsoNormal" style="color:#0E101A;mso-list:l0 level1 lfo1"><span style="font-size:12.0pt;font-family:"Times New Roman",serif">I$ - for Embench, the code size is up to 16KB, the number of dynamic instructions depends on clock frequency, but even for
 10MHz, it’s bigger than 80M instructions. Even if a missed penalty is 100s cycles, the impact is about 0.<o:p></o:p></span></li><li class="MsoNormal" style="color:#0E101A;mso-list:l0 level1 lfo1"><span style="font-size:12.0pt;font-family:"Times New Roman",serif">Branch Predictor – same as (1)<o:p></o:p></span></li><li class="MsoNormal" style="color:#0E101A;mso-list:l0 level1 lfo1"><span style="font-size:12.0pt;font-family:"Times New Roman",serif">D$ - data size is limited to 64K, assuming cache line is 32B, warming can save 2K misses. Assuming we have ~20% memory accesses,
 it’s 2K out of 16M; even with a missed penalty of 300 cycles, it’s only 0.03 cycles per memory access.<o:p></o:p></span></li></ol>
