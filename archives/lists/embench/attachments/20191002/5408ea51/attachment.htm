<p class="MsoNormal">I’m still testing the performance of Embench on various Arm devices.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">During testing, I encountered the ST (stats) test failing. This performs various double precision floating point operations.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">When using Arm’s fast math library on a device with no hardware floating point support, the test fails.
<o:p></o:p></p>
<p class="MsoNormal">(see <a href="https://developer.arm.com/docs/dui0774/i/compiler-command-line-options/-ffast-math-fno-fast-math">
https://developer.arm.com/docs/dui0774/i/compiler-command-line-options/-ffast-math-fno-fast-math</a>)<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">On debugging this further, this is due to a comparison mismatch in the verify_benchmark function.<o:p></o:p></p>
<p class="MsoNormal">If I modify the code with some debug printf statements:<o:p></o:p></p>
<p class="MsoNormal">                printf ("first check: %8.8e\n", fabs (expSumA - SumA));<o:p></o:p></p>
<p class="MsoNormal">                printf ("second check: %8.8e\n", fabs (expSumB - SumB));<o:p></o:p></p>
<p class="MsoNormal">                printf ("third check: %8.8e\n", fabs (expCoef - Coef));<o:p></o:p></p>
<p class="MsoNormal">                printf ("Coef %8.8e, expCoef %8.8e\n", Coef, expCoef);<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">                return (fabs (expSumA - SumA) < 1.0e13)<o:p></o:p></p>
<p class="MsoNormal">    && (fabs (expSumB - SumB) < 1.0e-13) && (fabs (expCoef - Coef) < 1.0e-17);<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">I see the following output:<o:p></o:p></p>
<p style="margin:0cm;margin-bottom:.0001pt">first check: 0.00000000e+00<o:p></o:p></p>
<p style="margin:0cm;margin-bottom:.0001pt">second check: 0.00000000e+00<o:p></o:p></p>
<p style="margin:0cm;margin-bottom:.0001pt">third check: 2.22044605e-16<o:p></o:p></p>
<p style="margin:0cm;margin-bottom:.0001pt">Coef 9.99900055e-01, expCoef 9.99900055e-01<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">(It also looks like there’s a code error here, since the first comparison is <1e13, the second is <1e-13. Shouldn’t the first also be 1e-13?)<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">The third check result about causes the test to fail since the third comparison looks for an error of less than 1e-17. Is there a particular reason for a tighter threshold on the third check? In fact the absolute difference is less than
 1e-17, it’s just that the comparison is done quickly and introduces some error. Can it be relaxed to 1e-13 to match the other comparisons? I don’t know where those numbers have been derived from.<o:p></o:p></p>
